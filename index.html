¿Qué es un Sistema de Control de Versiones?

"Una aplicación que les permite seguir y controlar los cambios realizados en los archivos de un proyecto (código fuente) de una manera estructurada y controlada."

¿Porqué es necesario?

  • Hace más fácil el deshacer errores, revertir versiones anteriores.
  • Hace más fácil el compartir código fuente con otros programadores sin crear conflictos.
  • Hace más fácil el distribuir cambios de un ambiente de desarrollo a producción.

Sistemas populares para Control de Versiones

  • SVN - SubVersioN
  • Mercurial
  • Git

¿Qué es Git?

  • Distributed Source Control.
  • Open source, gratis (GNU GPL V2).
  • Originalmente desarrollado por Linus Torvalds para el desarrollo del Linux Kernel en 2005.
  • Se enfoca en velocidad y eficiencia, diseño único, a veces da miedo y en ocasiones dificil de entender.

Qué es Git?

  • Graba snapshots, no diferencias.
  • Branching (ligero & rápido)
  • Merge automático de archivos.
  • Usado en proyectos personales o en proyectos grandes, y para cualquier tamaño de equipo.

Desarrollo distribuido

  • Cada "working directory" Git contiene el repositorio completo, historia y la capcidad completa de revisiones.
  • No son dependientes de un servidor central, y no necesitan estar conectados
  • Es extremadamente rápido - más que SVN, CVS y otros sistemas

Terminos utilizados y algunos comandos Git

  • Repository
